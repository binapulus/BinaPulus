🕐 Point-in-Time Recovery (PITR)
Firebase'in yerleşik yedekleme sistemi. Aktif edilirse son 7 güne kadar herhangi bir ana geri dönebilirsin. Veri silme veya bozulma durumunda hayat kurtarır.
📋 Nasıl Aktif Edilir?
  1. console.firebase.google.com → Projenizi seçin
  2. Sol menü → Firestore Database
  3. Üst sekme → Backups (Yedekler)
  4. "Point-in-time recovery" bölümünü bul
  5. "Enable" butonuna tıkla
  6. Onay kutusunu işaretle → Etkinleştir
⚠️ PITR Spark (ücretsiz) planda mevcut değildir. Blaze planına geçmen gerekir.
Blaze plan kullandıkça öde modelidir — az kullanımda aylık birkaç dolar civarı olur.

🔥 Firebase Güvenlik Kuralları
Firebase Console → Firestore → Kurallar sekmesine aşağıdaki kuralı yapıştır: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Ana yönetici config — sadece okunabilir
    match /anayonetici/{doc} {
      allow read: if false;
      allow write: if false;
    }

    // Binalar — kimliği doğrulanmış kullanıcılar
    match /binalar/{binaId}/veri/data {
      allow read, write: if request.time != null;
    }

    // Bildirim kuyruğu
    match /binalar/{binaId}/bildirimler/{doc} {
      allow read, write: if request.time != null;
    }
  }
}
⚠️ Daha sıkı kurallar için Firebase Auth entegrasyonunu tamamla.
📱 Push Bildirim — VAPID Key Kurulumu
  1. Firebase Console → Proje Ayarları → Cloud Messaging sekmesini aç
  2. "Web push certificates" bölümünde "Generate key pair" butonuna tıkla
  3. Oluşan anahtarı kopyala
  4. index.html dosyasında BURAYA_FIREBASE_VAPID_KEY_YAZIN yazan yeri bul ve değiştir
